以下是作业构想

二、游戏策划方案

1. 游戏名称

存活 7 天挑战

2. 游戏类型

单机、休闲

3. 游戏进行方式

随机事件

4. 游戏主题

如何让一个因疫情被封印在家的人存活下来（当然是靠运气啦（bushi

5. 游戏目的

留存特殊时期的记忆。

6. 游戏角色

1人（提供内置角色可供选择）

7. 游戏风格

类似电子宠物和旅行青蛙的休闲类小游戏，无需投入大量时间，但每次登录都有小惊喜（或者小失望 orz 因为角色可能死掉或离家出走溜去另一个城市），玩家只是作为一个观察者，观察着游戏中这个”薛定谔的人“。

8. 目标用户

疫情下精神疲惫的人群，可以在游戏中找到一丝乐趣（类似苦中作乐？大概也许）

9. 运行环境

Web 浏览器，手机浏览器（其中只有手机浏览器可分享）

10. 游戏背景故事

以疫情期间被封印的我和身边的朋友同事为原型，以一种半真实，戏谑的方式记录这段时间大家的各种遭遇以及见闻。

11. 游戏概述

【每天登录】及【随机用户交互】触发事件。包括但不限于每日核酸、抢菜、吃饭、学习、追剧、被公司或小区投喂蔬菜包、在群里和志愿者吵了一架、楼上半夜跳健身操被打扰到（之后遇到奇葩事情可能会相应添加游戏事件）

12. 人物设定

二选一，宅家期间，可能会不修边幅一点。

原本衣服采用条纹（有种被监禁的感觉），但碍于本次像素画尺寸较小，无法呈现条纹（需要尺寸稍大点吗？但可能会失去电子宠物的感觉）

三、技术选型

技术选型之类的，比较投机取巧了，直接问了B站首页风叶穿行游戏的制作者 D 大。主要是 unity 学习成本比较高，另外自己还不会 C#，从零开始会有点困难。如果用 js 游戏引擎，基本上就看看文档就行，不用学习新的语言。

大佬推荐了 phaser（内部是用 pixi 渲染的），这种封装好的游戏引擎对萌新友好，但它相对 unity 来说比较简陋，没有编辑器，没有官方支持的 ecs 架构，主要是用来渲染雪碧图的 2d 游戏引擎，因此可能需要接入一些第三方库（比如物理引擎的话可以用 matter.js，另外可能还需要各种模型文件、动画格式的支持。

大佬是直接用 Webgl 写了个游戏引擎，封装了一些常用功能如 ecs、timer、场景树、动画机等。我等草履虫望尘莫及 orz

如果自己用 webgl 写游戏引擎的话，渲染引擎可以试试 pixi（看了下 standalone 里好像有 pixi 诶，似乎是 21 年年度报告用到的，不愧是大佬）、three.js（这个好像用的人挺多的，似乎还可以渲染 3d） 和 babylon

综合下来我打算用官方推荐的项目模板 phaser3-typescript-parcel-template

本来想融合一下 react 的（因为想学习一下 orz），找了个 文档 有一些解决方案，但比较古早了，webpack 配置都更新了，有点跑不起来，暂时放弃，之后如果要用到状态管理的话再说

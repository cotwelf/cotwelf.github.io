一、JS 简介

【1】请简单描述一下你对 JavaScript 的理解。

JavaScript 是一种解释执行的高级编程语言，同时也是一门动态、弱类型的直译脚本语言，适合面向对象（基于原型）和函数式的编程风格。JavaScript 主要运行在一个内置 JavaScript 解释器的客户端中（例如 Web 浏览器），能够处理复杂的计算，操控文档的内容、样式和行为。能在客户端完成的操作（例如输入验证、日期计算等）尽量都由 JavaScript 完成，这样就能减少与服务器的通信，降低服务器的负载。完整的 JavaScript 由 3 部分组成：ECMAScript（提供核心语言功能）、DOM（文档对象模型） 和 BOM（浏览器对象模型）。

解释型语言：相对于编译型语言存在的，特点：源代码不是直接翻译成机器语言；运行：先翻译成中间代码，由解释器运行。比如 Python / JavaScript / Perl /Shell等都是解释型语言。 解释型语言：程序不需要编译，程序在运行时才翻译成机器语言，每执行一次都要翻译一次。 因此效率比较低。

弱类型语言：类型检查更不严格，比如 1 == ‘1’

静态类型：编译时报错；动态类型：运行时报错。

直译语言：又称直译式语言，是一种编程语言，它会将代码一句一句执行，而不需要经过编译器先行编译为机器码。这种编程语言需要利用解释器，在执行期，动态将代码逐句解释（interpret）为机器代码，或是已经预先编译为机器代码的子程序，之后再执行.

脚本语言：以简单的方式快速完成某些复杂的事情通常是创造脚本语言的重要原则，基于这项原则，使得脚本语言通常比 C 语言、C++ 语言或 Java 之类的系统编程语言要简单容易，也让脚本语言另有一些属于脚本语言的特性：（1）语法和结构通常比较简单；（2）学习和使用通常比较简单；（3）通常以容易修改程序的“解释”作为执行方式，而不需要“编译”；（4）程序的开发产能优于执行性能。

【2】什么是 DOM

DOM（文档对象模型）是针对 XML 但经过扩展用于 HTML 的应用程序编程接口（API）。DOM 把整个页面映射为多层节点结构（相当于把页面映射为一个对象，将其作为对象来对待，可以读到对象属性，同时带有一些操作该对象的方法）。HTML 或 XML 页面中的每个组成部分都是某种类型的节点，这些节点又包含着不同类型的数据。通过 DOM 创建的文档树形图，开发人员可以轻松自如地更改任何节点。

制定 Web 通信标准 W3C（万维网联盟）制定了 DOM 标准。

  • DOM1 级（1998） = DOM 核心 + DOM HTML，DOM 核心规定如何映射 XML 文档结构，DOM HTML 在 DOM 核心的基础上加以扩展，添加了针对 HTML 的对象和方法。DOM1 级的目标主要是映射文档结构。
  • DOM2 级扩充了 DOM 视图（这个不是很懂 orz）、DOM 事件（事件和事件处理接口，比如 onclick 等鼠标事件，onkeydown 等键盘事件等）、DOM 样式（基于 CSS 为元素应用样式的接口，如 document.getElementById("test").style）、DOM 遍历和范围（遍历和操作文档树的接口，如 parentNode()，firstChild() 等）等模块。
  • DOM3 级进一步扩展了 DOM，引入了以同一方式加载和保存文档的方法等。
  • DOM0 级标准 并不存在，它可能只是一个参考点，即 IE4 和 Netscape Navigator 4.0 最初支持的不同形式的 DHTML，并非 W3C 制定的 DOM 标准。
    DOM 并不只是针对 JS 的，很多别的语言也都实现了 DOM。如 SVG（可伸缩矢量图）。

DOM 兼容性：不同浏览器对 DOM 的兼容性不一，比如 IE8 只支持 1 级，IE9 才开始支持到 3 级，而且某些浏览器只支持某级的【部分】，在开发过程中可能需要做兼容。例如 IE8 之前可能只能通过操作 BOM 的方式（window.innerHeight）获取到视口高度，所以获取视口高度的兼容性写法最好是 const seeHeight = window.innerHeight || document.documentElement.clientHeight || document.body.clientHeight

【3】什么是 BOM

BOM（浏览器对象模型），它指的是把浏览器当做一个对象来对待，这个对象主要定义了与浏览器进行交互的方法和接口。BOM 的核心是 window，而 window 对象具有双重角色，它既是通过 js 访问浏览器窗口的一个接口，又是一个 Global（全局）对象。这意味着在网页中定义的任何对象，变量和函数，都作为全局对象的一个属性或者方法存在。window 对象含有location（浏览器所加载页面的详细信息，比如可以通过 window.location.href 获取到 url 等）、navigator（浏览器详细信息，比如 window.navigator.userAgent 获取当前设备（mac 还是 Windows）当前浏览器信息（Firefox 还是 Chrome，及其版本号）之类的）、screen 对象、history对象等子对象，并且 DOM 的最根本的对象 document 对象也是 BOM 的 window 对象的子对象。

【5】JavaScript 有哪些优势和劣势

【JS 的优势】

（1）JS 可在客户端替服务器分摊掉一些工作（例如数据验证、数学计算等），从而减少了和服务器的交互次数，降低了服务器压力。

早期拨号上网速度很慢，与服务器数据交互很慢，比如表单提交，JS 可以在前端过滤掉一些基本错误，例如信息未填写完整等。

（2）JS 比较容易上手，日常开发涉及的大部分语法都比较简单。

（3）用户能快速得到页面上的反馈。一些操作必须与服务器通信，例如提交数据、验证昵称重复等，这些操作会有无法避免的网络延迟，而其他在客户端运行的大部分操作，都能得到即时反馈。

（4）跨平台。JS 不会依赖操作系统（例如 Windows、IOS 等），只要有浏览器，就能正常执行。

（5）丰富界面、增强交互。JS 可以控制文档中的任何元素，定制元素的内容、样式或行为，用 JS 代替 CSS 可以实现复杂而多样的动画或特效（例如点击元素改变背景色）。

【JS 的劣势】

（1）兼容性低。各个浏览器对 JS 的支持程度不同，同一套脚本在不同浏览器中的执行结果会不同，有的完美执行，有的可能会提示错误。

（2）安全性低。由于 JS 在客户端运行，用户不仅可以查看 JS 源代码，还能嵌入恶意代码、替换或禁用脚本。

（3）中断运行。因为 JS 是一种直译语言，所以只要有一条出错，那么就会直接停止运行。

【6】以下对 JS 的理解中错误的是

A. JScript 和 JS 是等价的；

虽然 JS 和 JScript 都是 ECMAScript 的实现，但两者是由不同公司设计的，因此无法划等号。

ECMAScript: ECMA（欧洲计算机制造商协会）指定的技术委员会（来自微软等关注脚本语言发展的公司的程序猿们组成）共同定义的脚本语言标准。

ES5 是 ECMAScript 5 的缩写，也被称为 ECMAScript2009。 ECMAScript 标准的第六版是 ES6 或 ECMAScript6。 它也被称为 ECMAScript2015。

B. JS 脱离浏览器后就不能运行；

JS 主要运行在一个内置 JavaScript 解释器的客户端中，浏览器只是 ECMAScript 实现可能的宿主环境之一。

Node.js 是一个开源和跨平台的 JavaScript 运行时环境。Node.js 在浏览器之外运行 V8 JavaScript 引擎（Google Chrome 的内核），具体可以看 Node.js 篇（TODO）。

宿主环境不仅提供基本的 ECMAScript 实现，同时也会提供该语言的扩展（如 DOM，则利用 ECMAScript 的核心类型和语法提供更多更具体的功能，以便实现针对环境的操作），一遍语言与环境之间对接交互。ECMAScript 的其他宿主环境如 Node。可以理解为 ECMAScript 只是一种语法规范（脚本语言标准）

【*7】script 标签的常用属性

一般来说，script 标签放到 body 底部即可。也就是说，脚本会被延迟到整个页面都解析完毕后再运行。

浏览器在遇到 body 标签时才开始呈现内容，因为如果将 script 标签放到 head 中的话，意味着需要等全部 JS 代码都被下载、解析和执行完成后，才能开始呈现页面的内容。会使呈现页面出现延迟，延迟期间浏览器会是一片空白。当然也可以把 script 标签放到 head 中并添加 defer 属性，但只适用于外部脚本。

属性 类型 说明
async 可选。只对外部脚本有效。 可异步加载脚本。async等于新开一个进程单独执行
defer 可选。只对外部脚本有效。 脚本可以延迟到文档完全被解析和显示之后再执行。相当于把 script 标签放到 body 底部
src 可选。 要执行的外部文件路径
type 可选。 表示编写代码使用的脚本语言的内容类型。默认值 text/javascript

可以直接在 script 标签内写 JS 代码，也可以通过 src 属性引用外部文件。在引用外部 js 文件时，.js 扩展名可以省略。带有 src 属性的 script 标签内的 JS 代码不会被执行。

解释器会对 script 元素内部的代码从上到下依次解释，求值完毕以前，页面其余内容都不会被浏览器加载。

不能在 HTML 文档中省略 script 标签的结束标签，因为这种语法不符合 HTML 规范，而且也得不到某些浏览器（尤其是 IE）的正确解析。

【16】相等（==）和全等（===）运算符有哪些区别？

相等运算符用于比较两个操作数是否相等，操作数会进行类型转换。全等运算符用于比较两个操作数是否严格相等，操作数不会类型转换。即 1 == '1' 但 1 !== '1'。

开发过程中一律使用 ===，且比较之前需要将被比较的对象转换为同一类型，这样可以保持代码中数据类型的完整性，一定程度上避免出现 BUG。

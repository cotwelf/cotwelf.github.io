关于垃圾收集

（1）标记清除

主流算法，给当前不使用的值（该值是否可获得）加上标记，然后回收内存。

比如定义了一个定时器 let timer = setInterval(() => { console.log(2333) }, 1000)，在使用完毕后设置 timer = null，那么 setInterval(() => { console.log(2333) }, 1000) 这个值就无法通过访问作用域上的变量获得，就会被回收。

（2）引用计数

跟踪记录每个值被引用的次数。当声明了一个变量并将一个引用类型值赋给该变量时，则这个值的引用次数就是 1。如果同一个值又被赋给另一个变量，则该值的引用次数加 1。但它有个问题，就是 循环引用 的变量不会被清除

var element = document.getElementById("some_element");
var myObject = new Object();
myObject.element = element;
element.someObject = myObject;
// 正确做法，在页面卸载（销毁）之前，将其设为 null
myObject.element = null;
element.someObject = null;

类似的有，setTimeout 和 setInterval 在页面销毁前也需要清理，即将变量设置为 null，这种做法叫做解除引用，其作用是让值脱离执行环境，以便垃圾收集器下次运行时将其回收。

JS 会定时清理不用的变量以释放内存，比如局部变量会在函数执行结束后清理。

编写一个函数，让两个并不大的小数正数相乘

const accMul = (arg1, arg2) => {
  let m = 0 // 存放小数位数
  let s1 = arg1.toString()
  let s2 = arg2.toString()
  m = s1.split('.')[1].length + s2.split('.')[1].length
  let result = Number(s1.replace('.', '')) * Number(s2.replace('.', ''))
  return result / Math.pow(10, m)
}
accMul(0.03, 0.23)

由于 JavaScript 中的浮点数时基于 IEEE 754 标准来实现的，因此它的计算精度远远不如整数，例如 33.98 会显示成 33.9799999999。如果两个小数并不大，那么可以先将小数变成整数，再执行乘法运算，最后把小数点往左移动到指定位置，得到想要的结果。

注意！！Math.pow(base, exponent) 中的 exponent 不能是负数！！！

Math.pow(10, 4) // 输出：10000，正确
Math.pow(10, -4) // 输出：0.00009999999999999999，错误，应该是 0.001，也是浮点数计算精度问题

计算中最好把小数运算都换成整数运算，然后通过除以 pow 正数次方位移

统计字符串“xxxxyydda”中每个字母出现的次数

const counter = (str) => {
  let result = {}
  let keys = []
  for(let i = 0; i < str.length; i++) {
    if(keys.includes(str[i])) {
      result[str[i]] += 1
    } else {
      keys.push(str[i])
      result[str[i]] = 1
    }
  }
  return result
}

“3.00”。toFixed() 是内置对象 Number 的一个方法，用于格式化一个浮点数，返回数字的字符串形式。他的参数用于指定小数部分的显示位数，并对最后一位四舍五入。如果小数部分实际位数达不到指定位数，就用 0 补齐。表达式中的两个点号，第一个点被识别为小数点，第二个点被识别为成员访问运算符。相当于 3.0.toFixed(2)，计算结果为 “3.00”

按位非运算符（~）会先将所有值转换为整数，再以二进制表示，最后执行按位取反运算。

[] == ![] 得到的结果是（）

true。根据运算符的优先级规则：

  • 先计算逻辑非，![] 为 false，此时表达式为 [] == false
  • 空数组能转换为空字符串，表达式变为 "" == false
  • 当左操作数是字符串，右操作数是布尔值时，右边的布尔值要转为数字，所以 false 转为 0，表达式为 "" == 0
  • 当左操作数是字符串，右操作数是数字时，字符串要转为数字，所以 “” 转为 0，0 == 0 为 true

一句话总结：比较请同类型比较，否则会变得不幸 QAQ

执行下面的代码，调用 isPrototypeOf() 方法得到的结果是（），执行 instanceof 运算符得到的结果是（）

function child() {}
function ancestor() {}
child.prototype = ancestor;
var obj = new child()
ancestor.isPrototypeOf(obj);
obj instanceof ancestor;

isPrototypeOf() 方法用于检测【调用此方法的对象】是否存在于指定对象的原型链中，构造函数 child() 的原型指向构造函数 ancestor()，因此 ancestor() 在 obj 的原型链上，调用 isPrototypeOf() 方法得到的结果是 true。

instanceof 运算符是用于检测【构造函数的原型】是否存在于指定对象的原型链中，查看代码可知 ancestor() 的原型并不在 obj 对象的原型链上，因此执行 instanceof 运算符得到的结果是 false。

封装一个函数，能将字符串 “abcdef” 逆序为 “fedcba”

function reverse(str) {
  var result = ""
  var begin = str.length - 1
  for (var i = begin; i >= 0; i--) {
    result += str[i]
  }
  return result
}

错误处理的核心，是首先要知道代码里会发生什么错误。由于 JS 是松散类型的，而且也不会验证函数的参数，因此错误只会在代码运行期间出现。一般来说需要关注以下三种，它们可能会在特定模式下、或是没有充分检查备操作值的情况下出现。

  1. 类型转换错误
    • 使用全等 === 判断，避免比较前自动类型转换；
    • 控制流程语句前手动将条件转为 boolean（即使用 if(!!a){} 或者 if(typeof a !== undefined){} 代替 if(a){}）避免自动转换出现问题
  2. 数据类型错误
    • 用到处理特定类型值的函数之前，需要判断一下被处理值的类型，如
      function getQueryString(url) {
  if (typeof url === 'string') { // 这里如果不加判断，indexOf 可能会报错。
    var index = url.indexOf('?');
    if (index > -1) {
      return url.substring(index + 1)
    }
  }
  return '';
}

JSON 对象与 JS 对象的不同之处

JSON 只是一种数据格式，不从属于 JS，很多语言都有针对 JSON 的解释器。JSON 中不支持保存 undefined，其余和 JS 类似。

对象和数组通常是 JSON 数据结构的最外层形式（当然，这不是强制规定的），利用它们能够创造出各种各样的数据结构。

JSON 对象相比较 JS 对象有以下不同点：

  • 键值对中的 key 要加引号；
  • string 需要用双引号包裹；
  • 对象最后一个字段后不能加逗号；
  • 对象末尾不能加分号。
var obj = {
  "a": 1, // 单双引号或不加引号均可
  'b': {
    name: 'cotwelf' // string 也可以使用单引号或双引号包裹
  },
  c: 3, // 最后一个键值对之后可以加逗号也可以不加
}; // 末尾可加分号可不加分号

{
  "a": 1, // 属性必须用双引号
  "b": {
    "name": "cotwelf" // string 只能用双引号
  },
  "c": 3
} // 末尾没有分号

JSON.stringify() 与 JSON.parse() 使用过程中应该注意什么

在序列化 JS 对象时，所有函数的原型成员都会被忽略，不体现在结果中。值为 undefined 的属性也会被跳过。

注意，如果传给 JSON.parse() 的字符串不是有效的 JSON，该方法会抛出错误。

// 1. 简单值
JSON.parse("test") // Uncaught SyntaxError: Unexpected token e in JSON at position 1
JSON.stringify("test") // '"test"'
JSON.parse('"test"') // 'test'
JSON.parse(1) // 1
JSON.parse("null") // null
JSON.parse("true") // true
// 2. 对象
var obj = {
  "a": 1,
  'b': {
    name: 'cotwelf'
  },
  c: 3,
};
// 外面的单引号是为了不影响 JSON 内双引号的闭合
JSON.stringify(obj) // '{"a":1,"b":{"name":"cotwelf"},"c":3}'
// 3. 数组
JSON.stringify([233, "hi", true]) // '[233,"hi",true]'

JSON.stringify() 除了要序列化的 JS 对象外，还可接收两个参数，第一个参数是个过滤器，可以是一个数组，也可以是一个函数；第二个参数是一个选项，表示是否在 JSON 字符串中保留缩进。单独或组合使用这两个参数，可以更全面深入地控制 JSON 的序列化。

11. 将下面的对象序列化为 JSON 字符串，在序列化时去除 name 属性，并将数组的第一个元素变为 null

var json = {
  "name": "xiuwen",
  "age": 18,
  "like": ["anime", "game", "music"]
};
JSON.stringify(json, (key, value) => {
  if(key === 'name') {
    return undefined;
  }
  if (key === "0") {
    return undefined; // 数组中第一个值将变为 null
  }
  return value
})

关于 JSON.stringify(value[, replacer [, space]]) 中的后两个参数

replacer

var json = {
  "name": "xiuwen",
  "age": 18,
  "like": ["anime", "game", "music"]
};

replacer 是 function：

  • 如果返回一个 Number, 转换成相应的字符串作为属性值被添加入 JSON 字符串。
    let replacer = (key, value) => {
if (key === 'name') {
  return 2333
}
return value
}
JSON.stringify(json, replacer) // '{"name": 2333,"age":18,"like":["anime","game","music"]}'
  • 如果返回一个 String, 该字符串作为属性值被添加入 JSON 字符串。
    let replacer = (key, value) => {
if (key === 'name') {
  return 'xiuxiuxiu'
}
return value
}
JSON.stringify(json, replacer) // '{"name":"xiuxiuxiu","age":18,"like":["anime","game","music"]}'
  • 如果返回一个 Boolean, “true” 或者 “false” 作为属性值被添加入 JSON 字符串。
    let replacer = (key, value) => {
if (key === 'like') {
  return false
}
return value
}
JSON.stringify(json, replacer) // '{"name":"xiuwen","age":18,"like":false}'
  • 如果返回任何其他对象，该对象递归地序列化成 JSON 字符串，对每个属性调用 replacer 方法。除非该对象是一个函数，这种情况将不会被序列化成 JSON 字符串。
    let replacer = (key, value) => {
if (key === 'like') {
  return ["code", "eiga"]
}
if (key === "1") {
  return undefined // 因为上面的 like 的每个属性也调用 replacer 方法，所以 like[1] 变为 null
}
return value
}
JSON.stringify(json, replacer) // '{"name":"xiuwen","age":18,"like":["code",null]}'
  • 如果返回 undefined，该属性值不会在 JSON 字符串中输出。
    let replacer = (key, value) => {
if (key === 'like') {
  return ["code", "eiga"]
}
if (key === "age") {
  return undefined // 注意这里，key 非数组 index 的时候返回 undefined 才会删除属性，否则只会置 arr[index] 为 null
}
return value
}
JSON.stringify(json, replacer) // '{"name":"xiuwen","like":["code","eiga"]}'
    replacer 是 arr
    let replacer = ["like"]
JSON.stringify(json, replacer) // '{"like":["anime","game","music"]}'
    JSON.stringify() 一般和 JSON.parse() 结合 localstorage 使用，注意在 JSON.parse() 的 value 是 undefined、NaN、Infinity、函数、对象实例、变量或正则表达式时会报错，可以用 try catch。

另外，JSON 可以实现简单数据类型的深拷贝

function deepCopy(obj) {
  return JSON.parse(JSON.stringify(obj));
}

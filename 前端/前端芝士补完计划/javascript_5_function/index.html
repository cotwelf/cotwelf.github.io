定义函数的方式

fn1() // fn1, fn2
fn3() // Uncaught TypeError: fn3 is not a function
// 函数声明：创建了名为 fn1 的函数，fn1.name === 'fn1'
function fn1() {
  console.log(`fn1, ${fn2()}`)
}
function fn2() {
  return 'fn2'
}
// 函数表达式：创建了个匿名函数(匿名函数.name === '')赋值给变量 fn3, fn3.name === 'fn3'
var fn3 = function() {
  console.log('fn3')
}

函数实际上也是对象，而函数名实际上是指向对象的指针

怎样判断一个变量是否是对象，在我看来，可以 new 出来的都是对象

函数声明与函数表达式的区别在于：函数声明会提升，而函数表达式不会。在代码开始执行之前，解析器就已经通过一个名为 函数声明提升（function declaration hoisting） 的过程，读取并将函数声明添加到执行环境中（函数声明提升会先于变量提升）。对代码求值时，JS 引擎在第一遍会声明函数并将它们放到源代码树的顶部，变量提升结束之后才会开始执行。执行到函数所在的语句之前，变量 fn3 中不会保存有对函数的引用。

另外，由于代码执行之前函数声明会提升，以下做法在 ES 中是无效语法，会被 JS 引擎尝试修正，但不同浏览器修正后的结果可能会有差异

//不要这样做！
if(condition){
  function sayHi(){
    alert("Hi!");
  }
} else {
  function sayHi(){
    alert("Yo!");
  }
}
//可以这样做
var sayHi; // 注意，一定要先声明变量，否则在判断过程中定义和上面是一样的，毕竟代码执行前，变量声明会在函数声明后提升。
if(condition){
  sayHi = function(){
    alert("Hi!");
  };
} else {
  sayHi = function(){
    alert("Yo!");
  };
}

递归

我自己调用自己。使用函数表达式，这种方法在严格模式和非严格模式都可以。下面看个🌰

var fn1 = (function f(num){ // 创建了一个名为 f() 的声明函数，然后将它赋值给变量 fn1
  if (num <= 1){
    return 1;
  } else {
    return num * f(num-1);
  }
});
// 即使 fn1 被修改，函数的名字 f 仍然有效，所以递归调用照样能正确完成

这样做的原因是：直接调用 fn1 自己递归会有风险（如 fn1 被赋值给另一个函数 fn2，fn1 被修改，但执行另一个函数 fn2 时，它的内部还是调用的 fn1，这时 fn1 已经改变了其实）；使用 arguments.callee 严格模式下会报错。（以下为了解释原因，如已理解可跳过）

function fn1(num){
  if (num <= 1){
    return 1;
  } else {
    return num * fn1(num-1);
  }
}
// 但这样直接调用容易出问题，比如
var fn2 = fn1;
fn1 = null; // 罪魁祸首在这里，或者将 fn1 指针换成其他的引用对象，执行 fn2 的时候也会出问题
fn2() // Uncaught TypeError: fn1 is not a function

arguments.callee 是一个指向正在执行的函数的指针，我们用它来代替直接调用 fn1

function fn1(num){
  if (num <= 1){
    return 1;
  } else {
-   return num * fn1(num-1);
+   return num * arguments.callee(num-1); // 使用 arguments.callee 代替函数名，可以确保无论怎样调用函数都不会出问题。
  }
}

但在严格模式下，通过脚本访问 arguments.callee 会报错。

闭包

闭包是指有权访问另一个函数作用域中的变量的函数。创建闭包的常见方式，就是在一个函数内部创建另一个函数

function createFunctions() {
  var result = new Array();
  for (var i = 0; i < 10; i++){
    result[i] = function(){
      return i;
    };
  }
  return result;
}

这里 result 数组内所有的函数返回的都是 10，因为它们访问的是同一个 i。当 createFunctions() 执行结束后，result 是 function(){ return i} 函数组成的数组，当函数调用时，会在作用域链上寻找 i，首先在 createFunctions 中找到了 i = 10（for 循环已经在调用 createFunctions() 时执行结束，createFunctions()0 取到的是循环后的 i 值）。

function createFunctions() {
  var result = new Array();
  for (let i = 0; i < 10; i++){
    result[i] = function(num){
      return function(){
        return num;
      }
    }(i);
  }
  return result;
}
createFunctions()[3](); // 3

另外 var 定义的变量会提升，var 无法为块级作用域定义私有变量，const 和 let 可以（而且他俩定义的无法提升），所以也可以把 var 换成 let。

创建闭包必须维护额外的作用域，所以过度使用它们可能会占用大量内存。

this 的指向

函数被 new 修饰

this 绑定的是新创建的对象，例：var bar = new foo() 函数 foo 中的 this 就是一个叫 foo 的新创建的对象，然后将这个对象赋给 bar，这样的绑定方式叫【new 绑定】。

函数时使用 call, apply, bind 来调用的

this 绑定的是 call, apply, bind 的第一个参数。例：foo.call(obj), foo 中的 this 就是 obj，这样的绑定方式叫【显性绑定】。

函数时在某个【上下文对象】内被调用

this 绑定的是那个上下文对象，例：var obj = { foo: foo }; obj.foo(); foo 中的 this 就是 obj，这样的绑定方式叫【隐性绑定】。

如果都不是，即使用【默认绑定】

如: function foo() {...}; foo()，foo 中的 this 就是 window，严格模式下默认绑定到 undefined

var name = "The Window";
var object = {
  name : "My Object",
  getNameFunc : function(){
    var that = this; // 这里相当于把 object 的 this 赋值给 that（vue 就是这么操作的
    return function(){
      console.log(this, 'this') // Window {}
      console.log(that, 'that') // object {}
      return that.name;
    };
  }
};
alert(object.getNameFunc()());

因为浏览器的垃圾处理机制不同，如果闭包的作用域链中保存着一个 HTML 元素，那么就意味着该元素有可能无法被销毁，将会导致内存泄露。只要匿名函数存在，element 的引用数至少也是 1，因此它所占用的内存就永远不会被回收。做法是在函数的最后将保存 HTML 指针的变量设置为 null

立即执行函数

(function(){
  // 块级作用域
})()

将函数声明包含在一对圆括号中，表示它实际上是一个函数表达式。而紧随其后的另一对圆括号会立即调用这个函数。在匿名函数中定义的任何变量，都会在执行结束时被销毁。这种技术经常在全局作用域中被用在函数外部，从而【限制向全局作用域中添加多余的变量和函数】。

function(){
 //这里是块级作用域
}(); //出错！

这段代码会导致语法错误，是因为 JavaScript 将 function 关键字当作一个函数声明的开始，而函数声明后面不能跟圆括号。然而，函数表达式的后面可以跟圆括号。

apply() 和 call()

都是在特定的作用域中调用函数，能够扩充函数赖以运行的作用域，好处是对象不需要与方法有任何耦合关系。

window.color = "red"
var o = { color: "blue" }
function sayColor() {
  console.log(this.color)
}
sayColor(); // red
sayColor.call(this) // red
sayColor.call(window) // red
sayColor.call(o) // blue

// bind() 方法会创建一个函数的实例，低版本浏览器不支持
var objectSayColor = sayColor.bind(o)
objectSayColor() // blue

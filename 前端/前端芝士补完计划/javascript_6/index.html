函数声明与函数表达式的区别

fn1(); // 2233
fn2(); // Uncaught TypeError: fn2 is not a function

function fn1() {
  console.log("fn1"); // fn1
  fn3(); //fn3
}
var fn2 = function () {
  console.log("fn2");
};
function fn3() {
  console.log("fn3");
}

在代码开始执行之前，解析器就已经通过一个名为 函数声明提升（function declaration hoisting） 的过程，读取并将函数声明添加到执行环境中。对代码求值时，JS 引擎在第一遍会声明函数并将它们放到源代码树的顶部，变量提升结束之后才会开始执行。执行到函数所在的语句之前，变量 fn2 中不会保存有对函数的引用。

apply() 和 call()

都是在特定的作用域中调用函数，能够扩充函数赖以运行的作用域，好处是对象不需要与方法有任何耦合关系。

window.color = "red";
var o = { color: "blue" };
function sayColor() {
  console.log(this.color);
}
sayColor(); // red
sayColor.call(this); // red
sayColor.call(window); // red
sayColor.call(o); // blue

// bind() 方法会创建一个函数的实例，低版本浏览器不支持
var objectSayColor = sayColor.bind(o);
objectSayColor(); // blue

函数绑定

创建一个函数，可以在特定的 this 环境中以指定参数调用另一个函数

var handler = {
  message: "Event handled",

  handleClick: function (event) {
    alert(this.message + ":" + event.type);
  },
};
var btn = document.getElementById("my-btn");
btn.addEventListener("click", handle.handleClick); // 这样调用 handleClick 的话，this 指向的是 btn，而不是 handle，所以 handle.handleClick 会是 undefined
// 方法一：闭包
btn.addEventListener("click", function (event) {
  handle.handleClick(event);
});
btn.addEventListener("click", handle.handleClick.bind(handler)); // 这样调用 handleClick 的话，this 指向的是 btn，而不是 handle

函数柯里化

编写一个 add() 函数，能正常执行下面的代码，并能在控制台输出注释中的数字

var add = function () {
  let args = Array.prototype.slice.call(arguments);
  let inner = function () {
    args.push(...arguments);
    return inner;
  };
  // inner.toString = function () {
  //   return args.reduce(function(prev, cur) {
  //     return prev + cur
  //   })
  // }
  return inner;
};
console.log(add(1, 2)); // 3
console.log(add(1, 2, 3)); // 6
console.log(add(1)(2)); // 3

柯里化（currying）也叫不完全函数，是一种部分求值的技术，能把一个完整的函数调用分解成多次函数调用，每次只传入部分函数，返回一个接受剩下参数的函数，如此循环往复，直将所有参数传递过去，最后得出结果。

ES6 class 和构造函数有什么区别？

1. 开门见山：一句话概括

“class 是构造函数及其原型继承的语法糖，但增加了强制 new、自动严格模式、方法不可枚举、无提升等约束，使面向对象编程更安全、更规范。”

2. 分点对比（建议 3~5 个关键点，不要背课文）
可以按照记忆口诀：“调、提、枚、严、静、继” 展开：

调用方式：class 必须用 new，构造函数可当普通函数。

提升：构造函数声明提升；class 不提升，存在 TDZ。

枚举性：class 原型方法默认不可枚举；构造函数原型方法可枚举。

严格模式：class 内部自动严格模式；构造函数不自动。

静态方法：class 用 static 关键字；构造函数手动添加。

继承语法：class 用 extends + super；构造函数需要手动设置原型链和调用父类构造函数。

3. 补充深层细节（显示你理解本质）
new.target：两者都支持，但 class 中更常用于阻止直接实例化抽象类。

原型对象可写性：class 的 .prototype 不可被重新赋值（只读属性），但可以修改其属性；构造函数的 .prototype 可重新覆盖。

typeof 类名：仍然是 “function”，因为 class 本身就是一个构造函数包装器。

4. 指出“非语法糖”的部分（体现批判性思维）
虽然大多数行为可以用 ES5 模拟，但方法不可枚举、内部标记 [[IsClassConstructor]] 导致必须 new 调用等特性，无法用纯 ES5 完全实现。因此 class 不仅仅是语法糖，而是一种新的、更严格的构建对象的方式。

5. 使用建议（加分项）
“日常开发建议使用 class，因为它语法清晰、不易出错。但如果你需要兼容旧环境、或者需要动态修改原型链的高级技巧，构造函数仍然有用。另外，class 无法像函数那样被当作普通回调调用，有时在写高阶函数时需要注意。”

说几个你常用的 ES6+ 特性？

使用场景举例：

  • 可选链 ?. — 安全访问深层嵌套的 API 返回数据（res?.data?.list?.[0]）
  • 空值合并 ?? — 只有 null/undefined 时给默认值：value ?? 'default'，替代大量 && 判断
  • 解构赋值 — const { a, b: alias, ...rest } = obj
  • 扩展运算符 … - 合并数组/对象：[...arr1, ...arr2]、{ ...obj1, ...obj2 }
  • Promise.allSettled / Promise.any — 并发控制：allSettled 等待全部完成，any 只取最快成功的
  • Array.at(-1) - 负索引访问数组末尾：arr.at(-1) 等同 arr[arr.length-1]
  • structuredClone() — const copy = structuredClone(original) 替代 JSON.parse(JSON.stringify(original)) 做深拷贝，支持循环引用
  • WeakMap / WeakSet — 存对象关联数据且不阻碍垃圾回收（React Fiber 映射、DOM 元数据）
  • Proxy + Reflect — Vue3 响应式的基础
  • Symbol — 唯一标识符，用于对象私有属性、迭代器协议
  • Generator/Iterator — function*，配合 for...of

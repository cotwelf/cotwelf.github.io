1. 事件冒泡和事件捕获

事件冒泡：即事件开始时由最具体的元素（文档中嵌套层次最深的那个节点）接收，然后逐级向上传播到较为不具体的节点（文档）。

事件捕获：document 对象首先接收到 click 事件，然后事件沿 DOM 树依次向下，一直传播到事件的实际目标

“DOM2级事件”规定的事件流包括三个阶段：事件捕获阶段、处于目标阶段和事件冒泡阶段。事件捕获为截获事件提供了机会，然后是实际的目标接收到事件，可以冒泡阶段对事件做出响应。

关于冒泡和捕获，可以看 菜鸟教程的这个例子

2. 事件处理

假设有这样一个元素 <a id="a-node"></a> 需要我们为它绑定事件

// 阻止事件默认行为（如 a 标签的 href 重定向 url
const preventDefaultFn = function(event) {
  event.preventDefault()
}
// 阻止事件冒泡
const stopPropagationFn = function(event) {
  event.stopPropagation()
  console.log(`如果 event.currentTarget === event.target 说明该事件不是冒泡而来的`,event.currentTarget === event.target)
}
const divNode = document.createElement('div')
divNode.onclick = (event) => {
  // 如果不阻止事件冒泡，点击 a，父元素的事件也会被触发
  console.log(`也许你只是想点击 ${event.target.tagName}，但它的父元素 ${event.currentTarget.tagName} 也被点了其实`)
}
// 下面我们为 divNode 添加一个 a
const divChildA = document.createElement('a')
divChildA.innerText = 'click `div > a`'
divChildA.setAttribute('href', 'https://cotwelf.github.io/about/')
divChildA.addEventListener('click', preventDefaultFn)  // 阻止它的默认事件，它就不会打开链接
divChildA.addEventListener('click', stopPropagationFn)  // 阻止事件冒泡，这样父元素的点击事件就不会触发
divNode.appendChild(divChildA)
document.body.appendChild(divNode)

// divChildA.removeEventListener('click', stopPropagationFn)  // 可以用 removeEventListener 移除相应事件的绑定方法（这里只移除了 stopPropagationFn，但 preventDefaultFn 依旧生效）

注意，IE 的事件对象有所不同，如果用打包工具的话需要安装 babel-polyfill，可兼容 IE。但如果纯手写的话需要搜一下了，这里不做过多介绍因为不常用

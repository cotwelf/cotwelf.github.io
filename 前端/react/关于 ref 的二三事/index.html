关于 ref 的二三事

前端 React
  • Refs are an escape hatch to hold onto values that aren’t used for rendering. You won’t need them often.
  • A ref is a plain JavaScript object with a single property called current, which you can read or set.
  • You can ask React to give you a ref by calling the useRef Hook.
  • Like state, refs let you retain information between re-renders of a component.
  • Unlike state, setting the ref’s current value does not trigger a re-render.
  • Don’t read or write ref.current during rendering. This makes your component hard to predict.

参考文档： Referencing Values with Refs

ref 和 state 的区别

一句话总结：ref 和 state 一样用来存储数据，但 ref 的改变不会触发组件重新渲染，而 state 会。

const Component = () => {
    const clickTimesRef = useRef(0)
  useEffect(() => {
        console.log(`clickTimesRef.current: ${clickTimesRef.current}`)
    })
    return (
    <button
      onClick={() => {
        clickTimesRef.current += 1
        console.log(clickTimesRef.current)
        }}
    >{clickTimesRef.current}</button>
  )
}

可以看到每次 log 里的数字在增加，但 button 上的数值一直是 0，useEffect 里的 log 也只有在组件初次渲染时会 log 0，后续无法读取 ref 数值的变化。

但是！！state 的变化会导致页面渲染，渲染时会更新最新的 ref 的值（可能会带来一些混淆）例如上述代码加上 state 的更新，可以看到两个 div 的数值会一起变化，useEffect 里的内容同理

const Component = () => {
  const [clickTimes, setClickTimes] = useState(0)
    const clickTimesRef = useRef(0)
  useEffect(() => {
        console.log(`clickTimesRef.current: ${clickTimesRef.current}`)
        console.log(`clickTimes: ${clickTimes}`)
    })
    return (
    <>
      <div>{`clickTimesRef.current: ${clickTimesRef.current}`}</div>
      <div>{`clickTimes: ${clickTimes}`}</div>
      <button
        onClick={() => {
          clickTimesRef.current += 1
          setClickTimes(clickTimes + 1)
          console.log(clickTimesRef.current)
        }}
      >+1</button>
    </>
  )
}

什么时候用 ref

和非 React 相关 API 联动时（一般是浏览器 API）如：

  • 存储 timeout ID
  • 存储和操作 DOM 元素
  • 存储不需要被用来计算 JSX 的其他对象。

为什么 timeout ID 需要用 ref 存储

const Component = () => {
  const [timeoutId, setTimeoutId] = useState()
    const timeRef = useRef(null)
  const handleAlert = () => {
    timeRef.current = setTimeout(() => {
      alert(`timeRef.current: ${timeRef.current}`)
    }, 3000)
    setTimeoutId(timeRef.current)
  }
  const handleCancel = () => {
    console.log(timeRef.current)
    clearTimeout(timeRef.current)
  }
    return (
   <>
     <button
      onClick={handleAlert}
      >alert</button>
    <button onClick={handleCancel}>cancel</button>
    <div>{timeoutId}</div>
   </>
  )
}

可以看到 handleCancel 的 timeout ID 是即将 clear 的 ID

但如果把 ID 存储为普通参数，点击 alert 时存储到 timeoutId 的 ID，会在 3 秒后被 log 出来，但如果三秒内点击取消，可以看到 timer 是 0，因为 handleAlert 里 setTimeoutId 修改了 state 从而触发了组件渲染，组件重新渲染 timer 会被初始化，原先存储在 timer 上的 timeout ID 会被丢弃，但由于只是被丢弃到内存却并未清除，所以还是会在计时结束后执行 timeout 的方法（不过如果放弃设置 state，timer 是可以保存下来，但让组件完全不渲染太不现实，且多个组件嵌套的情况下无法仅在组件内部判断该组件渲染时机，所以直接用变量存储 timeout ID 是不推荐的）

const Component = () => {
  const [timeoutId, setTimeoutId] = useState()
    let timer = null
  const handleAlert = () => {
    timer = setTimeout(() => {
      alert(`timer: ${timer}`)
    }, 3000)
    setTimeoutId(timer)
  }
  const handleCancel = () => {
    console.log(timer)
    clearTimeout(timer)
  }
    return (
   <>
     <button
      onClick={handleAlert}
      >alert</button>
    <button onClick={handleCancel}>cancel</button>
    <div>{timeoutId}</div>
   </>
  )
}

存储 DOM 元素

  • Refs 是一个通用概念，但大多数情况下你会使用它们来保存 DOM 元素。
  • 你通过传递 <div ref={myRef}> 指示 React 将 DOM 节点放入 myRef.current。
  • 通常，你会将 refs 用于非破坏性操作，例如聚焦、滚动或测量 DOM 元素。
  • 默认情况下，组件不暴露其 DOM 节点。 你可以通过使用 forwardRef 并将第二个 ref 参数传递给特定节点来暴露 DOM 节点。
  • 避免更改由 React 管理的 DOM 节点。
  • 如果你确实修改了 React 管理的 DOM 节点，请修改 React 没有理由更新的部分。

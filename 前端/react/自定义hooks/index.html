什么？都 2202 了还有人不会写自定义 hooks？

前端 React

第一次见 React 函数式写法是 2020 年，因为团队几乎每人都会维护一个用以实现某个功能的组件，而这些组件在某一生命周期都有一些通用的功能，所以有大佬就自己写了 hooks 供大家使用。不得不说当时用惯 class 的我来说觉得 Hook 就是异端，反正官方文档也没说废弃 class，那就用着不好么？后来发现 Hook 是真滴方便，不仅可以用大佬自定义好的现成 hooks，还可以不用管 this 的问题（这两个对于我来说都比较难理解 orz，这次说自定义 hook，之后另外说说 this 吧~）。那么总当伸手党也不好，孩子也想知道怎么写大家都能用的 hooks！！

Hook 是 React 16.8 的新特性。它可以让你在不编写 class 的情况下使用 state 以及其他的 React 特性。

以我的理解，自定义 hooks，就是可以将 React 特性打包使用的一个特殊的“函数”。为什么说是一个“特殊的函数”呢？众所周知，组件内的函数中无法使用 hooks（为什么捏？就我的理解，因为 hooks 如果放在函数中，就无法使用 React 生命周期，因为不能确定函数的执行时机，可以 参看文档）

以下是一个非常简单的 demo，可以在 sandbox 尝试。（只看功能的话，以下的 useName 并不是很必要其实，只是为了测试使用方法而已，大佬轻喷 orz）

import React, { useEffect, useState } from "react";
import ReactDOM from "react-dom";

const useInputOpts = (name) => {
  const [value, setValue] = useState(name);
  const [style, setStyle] = useState({});
  const onChange = (event) => {
    setValue(event.target.value);
  };
  useEffect(() => {
    if (!value) {
      setStyle({ border: "1px red solid" });
    } else {
      setStyle({ border: "1px green solid" });
    }
  }, [value]);
  return {
    value,
    onChange,
    style
  };
};

const useName = (cb, deps) => {
  const [name] = deps;
  useEffect(() => {
    if (name) {
      cb();
    }
  }, [name, cb]);
};

const Component = () => {
  const [name, setName] = useState("");
  const inputOpts = useInputOpts("");
  useName(() => {
    alert(`hi, ${name}`);
  }, [name]);

  return (
    <div>
      <p>
        <span>please enter your name: </span>
        <input
          style={inputOpts.style}
          value={inputOpts.value}
          onChange={inputOpts.onChange}
          onBlur={() => setName(inputOpts.value)}
          onFocus={() => setName("")}
        />
      </p>
    </div>
  );
};

ReactDOM.render(<Component />, document.getElementById("root"));

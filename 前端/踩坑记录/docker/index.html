Docker 使用指北

前端 工具

因为公司 Web 环境太难配了，萌新非常不友好，每次拿到新电脑之后环境要配好久，还容易出问题，因此打算试一下 Docker，看看能不能实现环境自由(bushi)~ 为什么要使用 Docker 而不用虚拟机呢？因为 Docker 更轻量，它并不是模拟完整的操作系统，而是提供一个最小限度的程序运行的虚拟环境，对于硬盘巨小星人非常友好。

《Docker 中文文档》
《Docker 官网》

镜像和容器的关系，就像 类 和 实例，镜像是静态的定义，容器是运行的实体。

下载并运行 Docker

官网下载

容器操作

运行一个 nginx 服务: docker run -d -p 80:80 --name webserver nginx

停止: docker stop webserver

删除: docker rm webserver

镜像操作

拉取镜像: docker pull [选项] [Docker Registry 地址[:端口号]/]仓库名[:标签]

如: docker pull nginx:latest

删除镜像: docker image rm [选项] <镜像1> [<镜像2> ...]

可以通过 image ID 删除，如 docker image rm 501（501 是 id 前三位，能分辨就可），也可以通过 image name 删除，如 docker image rm nginx
查看本地已下载镜像: docker image ls

删除虚悬镜像: docker image prune （使用 ls 查看时发现有些镜像名变成了 < none >，大概是由于官方维护，将镜像名转移到新下载的镜像上，虚悬镜像可以随意删除）

如果在使用过程中发现拉取 Docker 镜像十分缓慢，可以配置 Docker 镜像加速器

dependencies

mkdir webserver

指定基础镜像

可以看到，web 的大多数依赖都是 php 的，所以我们选择 php 官方镜像 作为基础镜像

注意，下面这里有个坑，猜猜是什么坑呢？

FROM php:7.4-cli
RUN mv "$PHP_INI_DIR/php.ini-production" "$PHP_INI_DIR/php.ini" \
  && cd "$PHP_INI_DIR" \
  (...略略略，全部脚本见文档末尾...）
  && mkdir -p imagick \
    && pecl install xdebug

这里的坑就是，如果所有的指令都放到一个 RUN 里执行，会不知道执行到哪里了（终端如下图

（上述指令可以写成 shell 脚本，使用 COPY ./install_imagick.sh ./ 放到 docker 镜像里，但依旧不会解决上述问题）

FROM php:7.4-cli
COPY ./install_imagick.sh ./
RUN sh install_imagick.sh

所以我把有可能出错的地方单独 RUN，创建文件夹跳转目录等行为，可以合并到一个 RUN 脚本里。

这里也有个坑 orz，可以先预测一下会出现什么问题。。。

没戳，聪明的你一定发现了，万一下载速度巨慢。。。（别问我怎么知道的，现在我还在下 imagick 和 ImageMagick QAQ，所以打算下载好之后放在目录里，运行 Dockerfile 的时候 copy 进去（反正也不大，15M 左右）

FROM php:7.4-cli
RUN mv "$PHP_INI_DIR/php.ini-production" "$PHP_INI_DIR/php.ini"
RUN cd "$PHP_INI_DIR" \
  && curl -fsSL 'http://www.imagemagick.org/download/ImageMagick.tar.gz' -o imagemagick.tar.gz
RUN  mkdir -p imagemagick \
  && tar zxvf imagemagick.tar.gz imagemagick --strip-components=1 \
  && rm imagemagick.tar.gz \
  && cd imagemagick \
  && ./configure --prefix=.
RUN make && make install
RUN cd "$PHP_INI_DIR" \
  && curl -fsSL 'https://pecl.php.net/get/imagick-3.7.0.tgz' -o imagick.tgz \
  && mkdir -p imagick \
  && tar zxvf imagick.tgz imagick --strip-components=1 \
  && rm imagick.tgz \
  && cd imagick
RUN phpize
RUN ln -s "$PHP_INI_DIR/imagemagick/include/ImageMagick-7" "$PHP_INI_DIR/imagemagick/include/ImageMagick"
RUN ./configure --with-php-config="$PHP_INI_DIR/bin/php-config" --with-imagick="$PHP_INI_DIR/imagemagick"
RUN make && make install
RUN echo "extensions=imagick.so" > "$PHP_INI_DIR/php.ini"
RUN pecl install memcached \
  && pecl install redis \
    && pecl install xdebug

保存 Dockerfile，在 /webserver 目录下执行

docker build -t webserver:v1 .

可以在 Docker 的镜像仓库 Docker Hub 里找需要用的镜像。
